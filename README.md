# 코드 규칙

- 커밋 컨벤션
- 역할구분, 이슈  `[역할][이슈번호] 커밋종류: 내용` `예) [FE][#1] feat: 내용`

```markdown
- feat: 새로운 기능을 추가할 경우
- fix: 버그를 고친 경우
- docs: 문서 수정한 경우
- style: 코드 포맷 변경, 세미 콜론 누락, 코드 수정이 없는 경우
- refactor: 프로덕션 코드 리팩터링
- test: 테스트 추가, 테스트 리팩터링 (프로덕션 코드 변경 없음)
- chore: (코드의 수정 없이) 설정을 변경
```
